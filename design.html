# Axis Studios

# Work type

作者: @MisteryMonster

举例: https://rsshub.app/axis-studios/work/full-service-cg-production

路由: /axis-studios/:type/:tag?

参数:

  • type, 必选 -

    work, blog

  • tag, 可选 -

    文章内的 Work type URL: compositing, full-service-cg-production, vfx-supervision, realtime, art-direction, animation

文章内 Work type 指向的栏目地址，比如： https://axisstudiosgroup.com/work/full-service-cg-production (opens new window) 的 tag 为 full-service-cg-production，要注意的是 tag 和文章的目录是一样的。

有一些 tag 并不经常使用： Script, direction, production, design-concept 等等。

# Notefolio

# Works

作者: @nczitzk

举例: https://rsshub.app/notefolio

路由: /notefolio/:caty?/:order?/:time?/:query?

参数:

  • caty, 可选 -

    分类，见下表，默认为 all

  • order, 可选 -

    排序，可选 pick 指 Notefolio 精选，newest 指 最新，noted 指 知名，默认为 pick

  • time, 可选 -

    时间，可选 all 指 全部，day 指 最近24小时，week 指 最近一周，month 指 最近一个月，month3 指 最近三个月，默认为all

  • query, 可选 -

    关键词，默认为空

分类 韩文分类名 中文分类名
all 전체 全部
A7 공예 工艺品
J7 그래픽 디자인 平面设计
B7 디지털 아트 数字艺术
C7 영상/모션그래픽 视频 / 图形动画
D7 브랜딩/편집 品牌创建 / 编辑
E7 산업 디자인 工业设计
F7 UI/UX UI/UX
G7 일러스트레이션 插画
K7 타이포그래피 字体

# 用户作品

支持浏览器扩展 支持 RSSBud

作者: @junbaor

举例: https://rsshub.app/zcool/user/baiyong

路由: /zcool/user/:uid

参数:

  • uid, 必选 -

    个性域名前缀或者用户ID

例如:

站酷的个人主页 https://baiyong.zcool.com.cn 对应 rss 路径 /zcool/user/baiyong

站酷的个人主页 https://www.zcool.com.cn/u/568339 对应 rss 路径 /zcool/user/568339

# 发现

支持浏览器扩展 支持 RSSBud

作者: @nczitzk

举例: https://rsshub.app/zcool/discovery

路由: /zcool/discovery/:query?

参数:

  • query, 可选 -

    查询参数

在 发现页 (opens new window) 中选择查询参数后会跳转到对应搜索结果页面。此时地址栏 https://www.zcool.com.cn/discover/ 后的字段即为查询参数。

如：https://www.zcool.com.cn/discover/0!0!0!0!0!!!!2!-1!1 (opens new window) 查询参数为 0!0!0!0!0!!!!2!-1!1
